## 第一章：单片机概述

### 1.1 什么是单片机

单片机（Micro Controller Unit, MCU）是一种将计算机系统的主要功能模块集成在单一芯片上的微型计算机。它内部集成了：

| 模块 | 作用 | 对应计算机组成原理概念 |
|------|------|------------------------|
| **CPU** | 运算与控制核心 | 数据通路 + 控制器 |
| **RAM** | 数据存储器（运行时的变量、栈） | 主存 |
| **ROM/Flash** | 程序存储器（固化代码） | 辅存（但可直接取指执行） |
| **定时器/计数器** | 精确计时、PWM 生成 | 可编程间隔定时器（PIT） |
| **中断系统** | 对外部事件的实时响应 | 中断控制器 |
| **通信接口**（UART/I²C/SPI） | 与外部设备交换数据 | I/O 接口 |

### 1.2 51 单片机家族

- **STC89C52**（本学习笔记使用的芯片）
  - 位数：8 bit
  - RAM：512 Byte（注意：不是 KB）
  - ROM/Flash：8 KB
  - 内核：兼容 Intel 8051

**STC89C52 40脚 DIP 封装引脚图：**

     P1.1 ── 2│         │39── P0.0

     P1.3 ── 4│         │37── P0.2

   RST ──── 9│         │32── P0.7
P3.0/RXD── 10│         │31── EA/VPP

P3.2/INT0─ 12│         │29── PSEN

> **P0 口特别注意：** 开漏输出，没有内部上拉电阻！用作普通 GPIO 时必须外接 10kΩ 排阻。这也是为什么 HC6800-ES 开发板上 P0 口接了排阻。

> 所有与 8051 内核一致的单片机统称为**51 单片机**。8051 是 Intel 公司在 1980 年代推出的经典架构，至今广泛应用于嵌入式控制领域。

### 1.3 数制基础

电阻标称值的读法（三位数码标记法）：

```
102 → 10 × 10² = 1,000 Ω = 1 kΩ
473 → 47 × 10³ = 47,000 Ω = 47 kΩ
```

> 这与色环电阻的标记逻辑一致，来自于模电中电阻元件参数的标准化。

### 1.4 哈佛架构 vs 冯·诺依曼架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    哈佛架构 (Harvard)                            │
│                    (51 单片机采用)                                │
│                                                                  │
│  ┌──────────────┐        ┌──────────────┐                       │
│  │ Flash/ROM    │        │    RAM       │                       │
│  │ (程序存储器)  │        │ (数据存储器)  │                       │
│  │    8KB       │        │    512B      │                       │
│  └──────┬───────┘        └──────┬───────┘                       │
│         │                       │                                │
│    程序地址总线             数据地址总线                           │
│     (16-bit)                (8/16-bit)                           │
│         │                       │                                │
│         ▼                       ▼                                │
│  ┌──────────────────────────────────────┐                       │
│  │              CPU Core                │                       │
│  │    (两套总线，可同时取指和取数)       │                       │
│  └──────────────────────────────────────┘                       │
│                                                                  │
│  优点：取指令和访问数据可以并行，速度快                            │
│  缺点：硬件复杂，地址空间不统一                                    │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                  冯·诺依曼架构 (Von Neumann)                      │
│                (PC / ARM Cortex-A 采用)                          │
│                                                                  │
│  ┌──────────────┐                                               │
│  │ 统一存储器    │                                               │
│  │ 程序+数据    │                                               │
│  └──────┬───────┘                                               │
│         │                                                        │
│    统一地址/数据总线                                              │
│         │                                                        │
│         ▼                                                        │
│  ┌──────────────────────────────────────┐                       │
│  │              CPU Core                │                       │
│  │  (一套总线，取指和取数分时复用)       │                       │
│  └──────────────────────────────────────┘                       │
│                                                                  │
│  优点：硬件简单，地址空间统一                                      │
│  缺点：取指和数据访问不能同时进行 (冯·诺依曼瓶颈)                   │
└─────────────────────────────────────────────────────────────────┘
```

> **关键理解：** 51 单片机由于采用哈佛架构，访问程序存储器（Flash）和数据存储器（RAM）使用**不同的指令**：`MOVC` 读程序存储器，`MOV` 读数据存储器。这就是为什么 C51 中的 `code` 关键字可以将常量表放在 Flash 中 —— 编译器会自动使用 `MOVC` 指令来读取。

```
                        ┌── 嵌入式控制方向 ──┐

        │

                        ┌── 通用计算方向 ──┐
                        ▼                  ▼

        │

        │

        │

        │

```

- 51 汇编：面向单片机控制（IO 操作、位操作能力强）
- 8086 汇编：面向 16 位 CPU 和内存系统（分段寻址、DOS/BIOS 调用）

## 第二章：C51 语言基础

### 2.1 数据类型

#### 基本数据类型

| 数据类型 | 位宽 | 取值范围 | 说明 |
|----------|------|----------|------|
| `bit` | 1 位 | 0 或 1 | C51 特有类型，存放在位寻址区 |
| `sbit` | 1 位 | 0 或 1 | C51 特有类型，定义 SFR 中的某一位 |
| `char` | 8 位 | -128 ~ 127 | 有符号字符型 |
| `unsigned char` | 8 位 | 0 ~ 255 | **最常用**：GPIO 数据、串口数据、计数器 |
| `int` | 16 位 | -32768 ~ 32767 | 有符号整型 |
| `unsigned int` | 16 位 | 0 ~ 65535 | 大范围计数、定时器初值计算 |
| `short` | 16 位 | -32768 ~ 32767 | 在 Keil C51 中与 `int` 相同 |
| `unsigned short` | 16 位 | 0 ~ 65535 | 无符号短整型 |
| `long` | 32 位 | -2147483648 ~ 2147483647 | 占用 RAM 较多，慎用 |
| `unsigned long` | 32 位 | 0 ~ 4294967295 | 无符号长整型 |
| `float` | 32 位 | ~±1.175E-38 ~ ±3.4E38 | **一般不推荐频繁使用**：占资源多、运算慢 |
| `double` | 32 位 | 通常与 `float` 相同 | Keil C51 中 `double` 按 32 位处理 |
| `sfr` | 8 位 | 0 ~ 255 | C51 特有，定义 8 位特殊功能寄存器 |
| `sfr16` | 16 位 | 0 ~ 65535 | C51 特有，定义 16 位特殊功能寄存器 |

#### 类型的典型用途

| 类型 | 常见用途 |
|------|----------|
| `unsigned char` | GPIO 数据、串口数据、数组、标志变量 |
| `unsigned int` | 计数器、延时、定时器计算 |
| `bit` | 状态标志位，例如 `flag` |
| `sbit` | 定义 IO 口某一位，例如 LED、按键 |
| `sfr` | 定义特殊功能寄存器，例如 `P0`、`P1`、`TMOD` |
| `float` | 不推荐频繁使用（占用资源多、速度慢） |

> **计组视角：** 51 单片机是 8 位架构，ALU 一次只能处理 8 位数据。`int`（16 位）和 `long`（32 位）运算需要多条指令组合完成，因此使用 `unsigned char` 可以获得最高的代码效率。这也是嵌入式系统与桌面系统（32/64 位）编程习惯差异的根源 —— 在 51 上，变量位宽直接影响性能。

### 2.2 运算符

| 运算类型 | 运算符 | 作用 | 示例 |
|----------|--------|------|------|
| 算术运算 | `+` `-` `*` `/` `%` | 加/减/乘/除/求余 | `c = a + b;` |
| 自增/自减 | `++` `--` | 加 1 / 减 1 | `i++;` |
| 关系运算 | `>` `<` `>=` `<=` | 大小比较 | `if(a > b)` |
| 相等/不等 | `==` `!=` | 判断相等/不等 | `if(a == b)` |
| 逻辑运算 | `&&` `\|\|` `!` | 逻辑与/或/非 | `if(a && b)` |
| 位运算 | `&` `\|` `^` `~` | 按位与/或/异或/取反 | `P1 = P1 & 0xFE;` |
| 移位运算 | `<<` `>>` | 左移/右移 | `a = a << 1;` |
| 赋值 | `=` `+=` `-=` `*=` `/=` | 赋值/复合赋值 | `a += 1;` |

> **数电视角：** 位运算直接对应数字电路中的与门（&）、或门（|）、异或门（^）、非门（~）。移位运算本质上是移位寄存器 —— 用移位寄存器芯片（如 74HC595）可以将串行数据转换为并行输出，也可以用 74HC165 将并行输入转为串行数据。

### 2.3 基本语句

#### 51 单片机最常用的语句

| 语句类型 | 作用 | 示例 |
|----------|------|------|
| 变量定义语句 | 定义变量 | `unsigned char i;` |
| 赋值语句 | 给变量赋值 | `i = 10;` |
| IO 控制语句 | 控制单片机引脚 | `P1 = 0xFE;` |
| 位控制语句 | 控制某一个引脚 | `LED = 0;` |
| 条件判断语句 | 根据条件执行不同代码 | `if(key == 0)` |
| 多分支语句 | 多种情况选择 | `switch(mode)` |
| 循环语句 | 重复执行代码 | `while(1)` |
| 函数调用语句 | 调用函数 | `delay();` |
| 返回语句 | 从函数返回 | `return;` |
| 跳出语句 | 跳出循环或分支 | `break;` |
| 继续语句 | 跳过本次循环 | `continue;` |
| 空语句 | 什么都不做 | `;` |

#### 变量定义

```c
unsigned char i;       // 定义 8 位无符号变量
unsigned int count;    // 定义 16 位无符号变量
bit flag;              // 定义位变量，只能是 0 或 1
```

#### 赋值 vs 相等判断

```c
a = 10;    // 赋值：把 10 存入 a
a == 10;   // 判断：a 是否等于 10
```

#### IO 口控制

```c
P1 = 0xFE;  // 0xFE = 1111 1110 → P1.0 输出 0，P1.1~P1.7 输出 1
```

#### 位定义与位控制（sbit）

```c
sbit LED = P1^0;   // 定义 LED 连接到 P1.0
sbit KEY = P3^2;   // 定义按键连接到 P3.2

LED = 0;           // LED 输出低电平（低电平点亮时即为点亮）
LED = 1;           // LED 输出高电平（低电平点亮时即为熄灭）
```

> **C51 特殊语法注意：** `sbit LED = P3 ^ 5;` 是 Keil C51 的特殊语法，表示定义 P3.5 这一位。而普通 C 中 `x = P3 ^ 5;` 表示 P3 和 5 进行异或运算。两者含义完全不同。

#### 条件判断 if/else

```c
if (条件) {
    条件成立时执行;
} else {
    条件不成立时执行;
}

// 多条件判断
if (mode == 0) {
    P1 = 0xFE;
} else if (mode == 1) {
    P1 = 0xFD;
} else {
    P1 = 0xFF;
}
```

#### 多分支 switch

```c
switch(mode) {
    case 0:  P1 = 0xFE; break;
    case 1:  P1 = 0xFD; break;
    case 2:  P1 = 0xFB; break;
    default: P1 = 0xFF; break;
}
```

> 每个 `case` 后一般要加 `break;`，否则会继续往下执行（"fall-through"）。

#### 循环语句

```c
while(1) {           // 死循环 —— 单片机程序的标准模式
    // 主循环代码
}

for(i = 0; i < 10; i++) {  // 循环次数明确时使用
    P1 = i;
}

// 用于延时的空循环
for(i = 0; i < t; i++);    // 注意末尾的分号！

do {
    LED = 0;
} while(KEY == 0);          // 先执行一次，再判断
```

> **为什么单片机程序永远是死循环？** 单片机没有操作系统来管理程序退出，它上电后从复位向量开始执行，一旦 main 函数返回，PC（程序计数器）会跑到未定义区域，可能造成不可预知的行为。`while(1)` 保证了 CPU 始终在一个可控的循环中运行。对比通用计算机，OS 为每个程序分配时间片，程序退出后由 OS 回收资源 —— 单片机没有这些基础设施。

#### 函数

```c
// 函数定义
void delay(unsigned int t) {   // void: 无返回值  delay: 函数名  t: 形参
    while(t--);
}

// 函数调用
delay(1000);

// 有返回值的函数
unsigned char add(unsigned char a, unsigned char b) {
    return a + b;
}
```

| 概念 | 含义 |
|------|------|
| **形式参数（形参）** | 函数定义时的参数变量，调用前无实际值 |
| **实际参数（实参）** | 函数调用时传入的具体值 |
| **形参本质** | 函数内部的局部变量，调用期间存在 |

#### 指针

指针最核心的关系：

```c
int a = 10;
int *p = &a;

// a   → 变量本身，值是 10
// &a  → 变量 a 的地址
// p   → 指针变量，保存 a 的地址
// *p  → 通过地址访问 a 的值
```

> **短记：** 指针存地址，星号取内容。

> **计组视角：** 指针本质上就是内存地址。51 单片机的地址总线宽度决定了可寻址范围 —— 8051 有 16 位地址总线，可寻址 64KB。C51 中的指针与标准 C 的指针工作原理相同，但要注意 51 的哈佛架构使得程序存储器（Flash）和数据存储器（RAM）地址空间分离，访问不同空间需要使用不同的指令（MOVC vs MOV）。

### 2.4 模块化编程

**目的：** 将不同功能模块的代码分别放在 `.c` 文件中，在 `.h` 文件中提供对外接口的声明，提升代码的**可读性、可维护性、可扩展性、可移植性**。

```
传统方式：所有代码放在 main.c 中
模块化方式：每个功能模块独立为 .c + .h 文件对
```

**模块划分示意图：**
```
main.c
  ├── #include "Delay.h"     → delay.c
  ├── #include "LCD1602.h"   → LCD1602.c
  ├── #include "I2C.h"       → I2C.c
  ├── #include "AT24C02.h"   → AT24C02.c
  └── #include "Key.h"       → key.c
```

**文件职责：**
- `.c` 文件：函数和变量的**定义**（implementation）
- `.h` 文件：可被外部调用的函数和变量的**声明**（interface / declaration）

**头文件保护：**
```c
#ifndef __DELAY_H__
#define __DELAY_H__

void Delay(unsigned char i);

#endif
```

**使用规则：**
1. 任何自定义的变量和函数在被调用前必须有定义或声明
2. 使用到的 `.c` 文件必须全部加入工程参与编译
3. 使用到的 `.h` 文件必须放在编译器可搜索到的路径（工程根目录、安装目录、或自定义路径）

### 2.5 预编译指令

C 语言中以 `#` 开头的指令在**真正的编译之前**由预处理器处理：

| 指令 | 作用 |
|------|------|
| `#include` | 将指定文件内容复制到当前位置 |
| `#define` | 定义宏，后续出现的宏名将被替换为对应值 |
| `#ifndef ... #endif` | 条件编译：如果某个宏未定义，则编译中间的代码 |

### 2.6 sfr 与 sbit —— C51 中的寄存器操作

```c
sfr P0 = 0x80;        // 声明 P0 寄存器，物理地址 0x80
sbit P0_1 = P0 ^ 1;   // 声明 P0 寄存器的第 1 位
```

**位寻址 vs 不可位寻址：**

51 单片机的 SFR（特殊功能寄存器）中，地址以 0 或 8 结尾的寄存器是**可位寻址**的（硬件设计决定的）。不可位寻址的寄存器若要操作其中某一位而不影响其他位：

```c
P0 &= 0xFE;   // 将 P0.0 清零（按位与）
P0 |= 0x01;   // 将 P0.0 置一（按位或）
P0 ^= 0x01;   // 将 P0.0 翻转（按位异或）
```

> **数电视角：** 位寻址能力由芯片内部的地址译码电路直接决定。可位寻址意味着硬件为每个位提供了独立的地址线和读写控制，这是 8051 架构作为"面向控制"处理器的核心设计特征之一 —— 高效的单比特操作能力。

## 第四章：GPIO 与基本外设控制

### 4.0 从 C 代码到硬件：理解"控制"的本质

```
C 语言层
  │
  │  P2 = 0xFE;
  │
  ▼
编译器翻译
  │
  │  MOV  P2, #0FEH
  │
  ▼
CPU 执行
  │
  │  写信号 → 地址总线 0xA0 (P2 的物理地址)
  │  数据总线 = 0xFE
  │
  ▼
SFR 寄存器
  │
  │  ┌───────── P2 寄存器 (地址 0xA0) ─────────┐
  │  │  Bit7 Bit6 Bit5 Bit4 Bit3 Bit2 Bit1 Bit0│
  │  │   1    1    1    1    1    1    1    0  │
  │  └────┬────┬────┬────┬────┬────┬────┬────┘
  │       │    │    │    │    │    │    │    │
  ▼       ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼
IO 引脚   P2.7 P2.6 P2.5 P2.4 P2.3 P2.2 P2.1 P2.0
电平       H    H    H    H    H    H    H    L
          │    │    │    │    │    │    │    │
          ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼
LED 灯   灭   灭   灭   灭   灭   灭   灭   **亮**
(共阳)   (VCC=VCC, LED两端无压差 → 灭)
         (VCC=5V, P2.0=0V, LED两端有压差 → 亮)

总结: 你的 C 代码 P2 = 0xFE;
     → 被编译为 MOV P2, #0FEH 指令
     → CPU 在数据总线上放 0xFE，地址总线上放 0xA0
     → P2 寄存器锁存 0xFE
     → 8 个 IO 引脚的电平被硬件设置为对应的 1/0
     → 外接的 LED 因为压差而亮/灭
```

### 4.1 LED 控制

LED 连接到 **P2 寄存器**。通过配置 P2 即可控制 LED 的亮灭：

```c
P2 = 0xFE;  // 1111 1110 → P2.0 为低电平，若低电平点亮则对应 LED 亮
```

**硬件原理：** 在 HC6800-ES 开发板上，LED 通过限流电阻连接到 P2 口，另一端共阳（接 VCC），所以 **低电平点亮、高电平熄灭**。这是共阳接法中最常见的驱动方式。

**GPIO 口内部结构（简化）：**

```
           传统 8051 GPIO 引脚结构 (以 P1 口为例)

   VCC                        VCC
    │                          │
    └──┐                       └──┐
       │ 内部上拉电阻              │ 内部上拉电阻
       │ (~10kΩ)                 │ (~10kΩ)
       ├────────── 引脚 ────────┼──────────
       │                         │
    ┌──┴──┐                   ┌──┴──┐
    │ NMOS │ (输出低电平)      │ NMOS │
    └──┬──┘                   └──┬──┘
       │                         │
      GND                       GND

   输出模式：写 1 → NMOS 截止 → 引脚被上拉电阻拉高
             写 0 → NMOS 导通 → 引脚被拉到 GND
   
   输入模式：必须先写 1 (让 NMOS 截止)，
             然后读引脚电平
             
   注意：P0 口没有内部上拉电阻 (开漏输出)，
         用作普通 IO 时必须外接上拉！
```

> **数电视角：** 这就是一个典型的**推挽输出（Push-Pull）**变体 + **三态门**。NMOS 导通时输出低电平（强下拉），NMOS 截止时由上拉电阻提供高电平（弱上拉）。这就是为什么 51 的 IO 口高电平驱动能力弱（只能靠 ~10kΩ 上拉电阻），低电平驱动能力强（NMOS 直接接地）。驱动 LED 时用低电平点亮（灌电流方式），比高电平点亮（拉电流方式）更有效。

### 4.2 按键检测

**原理：** 按键未按下 → IO 口被上拉电阻拉高 → 读取为高电平；按键按下 → IO 口接地 → 读取为低电平。

```
按下 → 低电平（0）
未按下 → 高电平（1）
```

#### 按键消抖

机械按键在按下和释放时会产生抖动（约 5~20ms），导致电平多次跳变。如果不处理，单次按键可能被检测为多次。

> **模电视角：** 按键抖动本质上是机械触点的弹跳效应。硬件消抖（RC 滤波 + 施密特触发器）和软件消抖（延时检测）是两种互补方案。单片机的软件消抖本质上是用代码实现了一个低通滤波：如果信号在 20ms 内保持稳定才认为是有效电平。

**软件消抖流程：**

```
检测到低电平（疑似按下）
      ↓
延时 20ms（跳过抖动区）
      ↓
再次检测 → 仍是低电平 → 确认按下
      ↓
等待按键释放（while(P3_1 == 0);）
      ↓
延时 20ms（跳过释放抖动区）
      ↓
完成一次按键操作
```

```c
if (P3_1 == 0) {          // 检测到低电平
    Delay(20);             // 延时消抖
    if (P3_1 == 0) {      // 再次确认
        // 按键处理逻辑
    }
    while(P3_1 == 0);     // 等待释放
    Delay(20);             // 释放消抖
}
```

### 4.3 按键控制 LED 移位

通过按键控制 LED 灯的位置左右移动：

- **左移：** `NUMled` 从 0→7，到达边界 7 时返回 0
- **右移：** `NUMled` 从 7→0，到达边界 0 时返回 7

```c
P2 = ~(0x01 << NUMled);  // 将 0x01 左移 NUMled 位后取反，点亮对应 LED
```

## 第五章：中断系统

### 5.1 中断的基本概念

**中断系统**使 CPU 具有对外界紧急事件的**实时处理能力**。中断发生时，CPU 暂停当前主程序，保存现场，跳转到中断服务函数执行，执行完毕后恢复现场，继续原程序。

**中断处理流程：**

```
                    ┌─────────────┐
                    │ 主程序执行   │
                    │ while(1){   │
                    │   LED=0;    │
                    │   delay();  │ ◄── 硬件自动保存 PC 到堆栈
                    │   LED=1;    │
                    │ }           │
                    └──────┬──────┘
                           │
                    中断触发 (硬件事件)
                    (定时器溢出/外部引脚/串口接收)
                           │
                    ┌──────▼──────┐
                    │ 检查中断使能 │
                    │ EA && ET0?  │
                    └──────┬──────┘
                           │ 是
                    ┌──────▼──────┐
                    │ 保存现场     │ ← PC 自动入栈，PSW/ACC 等需手动
                    │ 跳转 ISR    │ ← PC = 中断向量地址 (如 0x000B)
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │ 执行中断服务 │
                    │ 函数 ISR    │
                    │ interrupt 1 │
                    │ { ... }     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │ 恢复现场     │ ← RETI 指令: 恢复 PC，释放优先级
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │ 返回主程序   │
                    │ 从中断点继续 │
                    └─────────────┘

优先级规则：高优先级中断可打断低优先级 ISR
          同级中断不能相互打断
```

> **计组视角：** 中断是计算机体系结构中最核心的 I/O 机制之一。与轮询（Polling）相比，中断跳过了"CPU 反复查询外设状态"的浪费过程。在 51 单片机中，中断向量表位于程序存储器的低地址区（0x0003~0x002B），每个中断源有固定的入口地址。当外设触发中断时，硬件自动将 PC 设置为对应的向量地址 —— 这本质上是一个"硬件实现的函数指针调用"。

### 5.2 51 单片机的中断资源

51 单片机共有 **8 个中断源**，**4 个优先级**：

| 中断号 | 中断源 | 中断向量地址 | 说明 |
|--------|--------|--------------|------|
| 0 | 外部中断 0 | 0x0003 | P3.2 引脚触发（INT0） |
| 1 | 定时器 0 | 0x000B | TF0 溢出触发 |
| 2 | 外部中断 1 | 0x0013 | P3.3 引脚触发（INT1） |
| 3 | 定时器 1 | 0x001B | TF1 溢出触发 |
| 4 | 串口中断 | 0x0023 | RI 或 TI 触发 |

**中断优先级：** 4 个级别（由 IP 寄存器配置），高优先级可打断低优先级的中断服务（中断嵌套）。

### 5.3 中断服务函数

```c
void Timer0_Routine() interrupt 1 {
    // interrupt 1 表示这是定时器 0 的中断服务函数
    static unsigned int T0Count = 0;  // static 保证变量在函数退出后保留
    // ...
}
```

> **注意：**
> - `interrupt n` 中的 n 是中断号，不是优先级
> - 中断服务函数**不需要手动调用**，由硬件自动触发
> - `static` 局部变量在中断函数中非常重要 —— 没有它，每次进入中断变量都会重新初始化

### 5.4 使用外部中断

通过配置寄存器实现外部中断（红外遥控会用到）：

```
IT0：触发方式（0=低电平触发，1=下降沿触发）
EX0：外部中断 0 使能
EA：总中断使能
```

## 第六章：定时器与计数器

### 6.1 定时器的工作原理

**寄存器是连接软件与硬件的媒介。** 每个寄存器既是一段可读写的存储器，又通过背后的导线连接着实际的数字电路。配置寄存器就相当于操作一个复杂机器的控制面板。

**定时器内部计数流程：**

```
                      ┌─────────────────────┐
      晶振            │   定时器 0 (16bit)   │
   ┌──────────┐       │                      │
   │ 12MHz    │──┐    │   ÷12  ┌──────────┐  │
   │ 11.0592  │  │    │  ────▶│  TL0 TH0 │  │
   └──────────┘  │    │  1MHz  │ (计数器)  │  │
                 │    │        └────┬─────┘  │
                 │    │             │        │
    TR0 = 1 ─────┼────┼─────────────┤        │
    (启动)       │    │             ▼        │
                 │    │    从初值开始加 1    │
    C/T0 = 0 ────┼────┤   (定时模式)         │
    (定时/计数)  │    │             │        │
                 │    │       加到 0xFFFF    │
                 │    │             │        │
                 │    │      再 +1 溢出      │
                 │    │             │        │
                 │    │      ┌─────▼──────┐  │
                 │    │      │  TF0 = 1   │  │
                 │    │      │  溢出标志   │  │
                 │    │      └─────┬──────┘  │
                 │    │            │         │
    ET0 = 1 ─────┼────┼────────────┤         │
    EA  = 1 ─────┼────┼────────────┤         │
                 │    │            ▼         │
                 │    │    ┌──────────────┐  │
                 │    │    │ 产生中断请求  │  │
                 │    │    │ PC → 0x000B  │  │
                 │    │    └──────────────┘  │
                 │    └──────────────────────┘
                 │
          内部机器周期 = 晶振 / 12
          1 个计数 = 1 个机器周期
```

> **每次溢出后 TH0/TL0 会被清零，所以中断服务函数里必须手动重装初值！**
> 这也是为什么每个 Timer0 中断函数开头都要写 `TH0 = ...; TL0 = ...;`

> **FPGA 视角：** 从硬件描述语言的角度看，定时器本质上是一个**带使能和预置数的计数器**。用 Verilog 描述：
> ```verilog
> always @(posedge clk) begin
>     if (reset) count <= 0;
>     else if (enable) count <= count + 1;
> end
> ```
> 51 单片机中的定时器硬件就是这样一个计数器，只是它被固化在硅片上，通过 SFR 来配置参数。

### 6.2 定时器的核心公式

**最核心公式：**

```
指令执行时间 = 指令周期数 / CPU主频
```

**传统 51 单片机（12T 架构）：**

```
机器周期 = 12 / 晶振频率
指令时间 = 机器周期数 × 机器周期
```

**示例（12MHz 晶振）：**

```
12MHz / 12 = 1MHz  →  1 次计数 = 1 μs
1 个机器周期 = 1 μs
普通指令大约 1 μs
复杂指令大约 2~4 μs
```

**11.0592MHz 晶振（精准波特率用）：**

```
1 个机器周期 ≈ 1.085 μs
```

> 11.0592MHz 这个看似奇怪的频率是因为它可以被常见波特率（9600、19200 等）整除，从而产生精确的波特率时钟。这体现了模电中晶振选型与实际需求的折中。

### 6.3 1T / 6T / 12T 单片机

| 类型 | 含义 |
|------|------|
| 12T | 12 个时钟周期 = 1 个机器周期（传统 8051） |
| 6T | 6 个时钟周期 = 1 个机器周期 |
| 1T | 1 个时钟周期 = 1 个机器周期（现代 STC 芯片支持） |

### 6.4 定时器初始化代码

```c
void Timer0_Init(void) {
    TMOD &= 0xF0;    // 清除定时器 0 配置
    TMOD |= 0x01;    // 定时器 0 模式 1（16 位定时器）

    TH0 = 64536 / 256;  // 装载初值高 8 位
    TL0 = 64536 % 256;  // 装载初值低 8 位

    TF0 = 0;  // 清除溢出标志
    TR0 = 1;  // 启动定时器 0
    ET0 = 1;  // 允许定时器 0 中断
    EA  = 1;  // 开总中断
}
```

**初值计算（12MHz 晶振，定时 1ms）：**

```
计数次数 = 定时时间 / 计数周期 = 1000μs / 1μs = 1000
初值 = 65536 - 1000 = 64536
TH0 = 64536 / 256 = 252
TL0 = 64536 % 256 = 24
```

### 6.5 软件累计实现长定时

16 位定时器最大只能计 65536 次（约 65ms@12MHz），要实现秒级定时需要**软硬结合**：

```
硬件定时器 → 每 1ms 产生一次中断（硬）
软件变量   → 累计 1000 次中断 = 1 秒（软）
```

```c
void Timer0_Routine() interrupt 1 {
    static unsigned int T0Count = 0;

    TH0 = 64536 / 256;   // 重装初值
    TL0 = 64536 % 256;

    T0Count++;
    if (T0Count >= 1000) {
        // 每 1000 次 = 约 1 秒
        P2_0 = !P2_0;    // 翻转引脚
        T0Count = 0;
    }
}
```

### 6.6 定时器模式 vs 计数器模式

| 模式 | C/T 位 | 计数来源 | 加 1 条件 |
|------|--------|----------|-----------|
| **定时器** | `C/T = 0` | 内部机器周期 | 每个机器周期加 1 |
| **计数器** | `C/T = 1` | 外部引脚 T0/T1 脉冲 | 外部脉冲有效沿加 1 |

> **不同平台的计数器加一规律不同：**
> | 平台 | 计数器加一条件 |
> |------|----------------|
> | 传统 8051 (12T) | 每个机器周期加 1（定时模式） |
> | STC 1T 单片机 | 可能是每个时钟周期加 1 |
> | STM32 | 由 APB 时钟和预分频器决定 |
> | FPGA | 由接入的时钟决定 |
>
> **本质规律：** 计数器每收到一个有效计数时钟，就加 1。至于这个时钟来自哪里，由具体平台和配置决定。

## 第七章：显示设备

### 7.1 LED 数码管

**工作原理：** 通过时分复用减少 IO 口的使用。数码管分为**共阴**和**共阳**两种接法。

#### 时分复用

| | 不使用复用 | 使用时分复用 |
|------|------------|--------------|
| 连接方式 | 每一路信号单独通道 | 多路信号共享一个通道 |
| 硬件资源 | 多 | 少 |
| 成本 | 高 | 低 |
| 控制 | 简单 | 复杂 |

**时分复用的关键技术参数：**

| 问题 | 含义 |
|------|------|
| 时间片长度 | 每个对象占用资源多久 |
| 切换速度 | 切换太慢会感觉卡顿或闪烁 |
| 同步问题 | 接收端需知道当前时间片属于谁 |
| 带宽要求 | 共享通道速度要足够快 |
| 控制逻辑 | 需要计数器、选择器、调度逻辑 |

#### 3-8 译码器（74HC138）

通过 3-8 译码器可以减少位选的 IO 口数量：3 根线控制 8 个数码管的位选。

> **数电视角：** 74HC138 是典型的 3-to-8 译码器，输入 3 位二进制地址，输出 8 根线中对应的一根为低。这本质上是组合逻辑电路。在数码管扫描中，74HC138 承担的是"选通"功能——决定当前哪个数码管被点亮。

#### 数码管驱动方式

| 方式 | IO 数量 | 特点 |
|------|---------|------|
| **单片机直接扫描** | 3+8=11 根 | 硬件简单，占用大量 CPU 时间 |
| **专用驱动芯片 TM1640** | 2 根（I²C） | 省 IO，不占 CPU |
| **74HC595 扩展** | 3 根（SPI 类） | 可级联，适合作坊级扩展 |

#### 消影处理

问题：位选→段选→位选→段选的切换过程中可能出现"串位"（上一位置的段码短暂显示在下一位置）。

解决方案：**位选 → 清零 → 段选 → 位选**

```c
void NixieDisplay(unsigned char Location, unsigned char Number) {
    // 1. 位选
    P2 = LocationMask;
    // 2. 段选
    P0 = SegCode[Number];
    Delay(1);
    // 3. 清零（消影）
    P0 = 0x00;
}
```

### 7.2 LCD1602 液晶显示器

**概述：**
- 字符型液晶显示模块，可显示 ASCII 标准字符和内置特殊字符
- 显示容量：16×2 个字符（每行 16 个，共 2 行）
- 每个字符为 5×7 点阵
- 支持 8 个自定义字符

**引脚定义：**

| 引脚 | 名称 | 功能 |
|------|------|------|
| VSS | 地 | 电源地 |
| VDD | 电源 | +5V |
| VO | 对比度 | 外接电位器调节 |
| RS | 数据/指令 | 1=数据，0=指令 |
| RW | 读/写 | 1=读，0=写 |
| E | 使能 | 1=有效，下降沿执行 |
| D0~D7 | 数据 | 8 位数据总线 |
| A | 背光正 | 接 +5V |
| K | 背光负 | 接 GND |


**字符与编码：**
- 字符按照 ASCII 码表建立从数字到字符的映射
- 字符数组：`unsigned char str[] = "ABC";`（末尾自动加 '\0'）
- 字符串 = 字符数组 + 结束标志 '\0'

### 7.3 LED 点阵屏

**分类：**
- 按颜色：单色、双色、全彩
- 按像素：8×8、16×16 等（大面积屏由小模块拼接而成）
- 按接法：共阳、共阴

**显示原理：** 与数码管类似，需要进行**逐行或逐列扫描**才能实现所有 LED 同时显示的效果。

#### 74HC595 扩展 IO

**74HC595 是串行输入并行输出的移位寄存器：**
- 3 根信号线输入（SER 数据、SCK 时钟、RCK 锁存）
- 8 根并行输出
- 多片级联后可扩展到 16、24、32 位输出

> **FPGA 视角：** 74HC595 就是硬件实现的串并转换器（SIPO）。如果你用 Verilog 写：
> ```verilog
> always @(posedge SCK) begin
>     shift_reg <= {shift_reg[6:0], SER};
> end
> ```
> 这就是 74HC595 内部逻辑的 HDL 等价描述。理解这一点后，你会意识到：单片机用软件模拟的时序，FPGA 用硬件并行执行——速度差几个数量级。


#### 三极管驱动

通过三极管提高电流驱动能力。单片机 IO 口输出电流有限（通常 <20mA），无法直接驱动大电流负载。

> **模电视角：** 这里的三极管工作在**开关状态**（截止-饱和），而非放大状态。NPN 管：基极高电平 → 集电极-发射极导通（饱和）→ LED 回路有电流 → 亮。这比让三极管工作在放大区简单得多，但需要注意基极限流电阻的选择，确保三极管能进入深度饱和。

## 第八章：串口通信

### 8.1 概述

串口（UART）是最广泛使用的通信接口之一，**成本低、使用简单、线路少**。51 单片机内部自带 UART（通用异步收发器）。

**基本连接：**
- TXD（发送）与 RXD（接收）交叉连接
- 只需单向传输时，可直连一根通信线
- 电平不一致时需增加电平转换芯片

### 8.2 电平标准

| 标准 | 逻辑 1 | 逻辑 0 | 说明 |
|------|--------|--------|------|
| **TTL** | +5V | 0V | 晶体管-晶体管逻辑，单片机原生信号 |
| **RS232** | -3V~-15V | +3V~+15V | 传统 PC 串口，需 MAX232 电平转换 |
| **RS485** | A-B 压差 +2~+6V | A-B 压差 -2~-6V | 差分信号，抗共模干扰好，可远距离传输 |

> **模电视角：** 差分信号（RS485）的本质是通过传输一对互补信号来抵抗共模噪声。接收端只关心两根线的**差值**，不关心绝对电平。这与运放中差分放大器的原理相同 —— 共模信号被抑制，差模信号被放大。


### 8.3 常用通信接口对比

| 接口 | 信号线 | 双工方式 | 同步/异步 | 拓扑 | 典型速度 |
|------|--------|----------|-----------|------|----------|
| **UART** | TXD, RXD | 全双工 | 异步 | 点对点 | ~115200 bps |
| **I²C** | SCL, SDA | 半双工 | 同步 | 多设备总线 | 100k/400k/3.4M |
| **SPI** | SCLK, MOSI, MISO, CS | 全双工 | 同步 | 多设备（独立 CS） | ~10 Mbps |
| **1-Wire** | DQ | 半双工 | 异步 | 多设备总线 | ~16 kbps |
| **CAN** | CANH, CANL | 半双工 | 异步 | 多设备总线 | 1 Mbps |
| **USB** | D+, D- | 半双工 | 同步 | 点对点（Hub 级联） | 12M/480M/5G+ |

### 8.4 通信基本概念

| 概念 | 含义 |
|------|------|
| **全双工** | 双方可同时收发 |
| **半双工** | 双方可分时收发（共享一根数据线） |
| **单工** | 只能单向传输 |
| **同步** | 双方靠共同时钟线约定速率 |
| **异步** | 双方各自约定速率（靠波特率匹配） |
| **总线** | 连接多个设备的共享数据传输线路 |

### 8.5 51 单片机的 UART

- 只有 **1 个** UART 资源
- 4 种工作模式：
  - **模式 0**：同步移位寄存器
  - **模式 1**：8 位 UART，波特率可变（**最常用**）
  - **模式 2**：9 位 UART，波特率固定
  - **模式 3**：9 位 UART，波特率可变

**串口通信帧格式：**

```
起始位(1) + 数据位(8/9) + 校验位(可选) + 停止位(1)
```

| 参数 | 含义 |
|------|------|
| **波特率** | bits per second，每秒传输的位数 |
| **校验位** | 奇校验（使 1 的总数为奇）/ 偶校验（使 1 的总数为偶） |
| **停止位** | 帧间间隔标志 |

**波特率计算（11.0592MHz 晶振）：**

使用定时器 1 的溢出率来产生波特率：

> 11.0592MHz 晶振的选择恰好是为了让波特率能被精确分频，这体现了数字系统中时钟频率与协议参数的精准配合。

## 第九章：I²C 总线与 EEPROM

### 9.1 I²C 总线简介

I²C（Inter-IC Bus）由 Philips 公司开发，是一种通用的两线串行总线。

- **信号线：** SCL（串行时钟）、SDA（串行数据）
- **模式：** 同步、半双工、带数据应答
- **优点：** 统一通信标准，缩短芯片设计周期，提高系统稳定性

**常见 I²C 设备：** 0.96" OLED 屏幕、DS3231 RTC、MPU6050 陀螺仪、AT24C02 EEPROM 等。

### 9.2 I²C 电路规范

1. 所有 I²C 设备的 SCL 连在一起，SDA 连在一起
2. SCL 和 SDA 必须配置为**开漏输出**模式
3. SCL 和 SDA 各接一个**上拉电阻**（通常 4.7kΩ）

```
     VCC
      |
    上拉电阻
      |
    引脚 OUT
      |
    NMOS
      |
     GND
```

> **数电视角：** 开漏输出 + 上拉电阻的组合实现了**"线与"（Wired-AND）**功能：任何一个设备将总线拉低，整个总线就变低；所有设备都释放时，上拉电阻将总线拉高。这种设计精妙地解决了多主机通信时的冲突问题 —— 两个设备同时发送时不会产生短路，因为这相当于两个 NMOS 并联接地。

### 9.3 I²C 时序

**起始条件（START）：** SCL 高电平时，SDA 从高→低
**终止条件（STOP）：** SCL 高电平时，SDA 从低→高

**I²C 时序波形：**

```
SCL:  ─┐     ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐     ┌──
       │     │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │     │
       └─────┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └─
       ┌──┐          ┌──┐     ┌──┐     ┌──┐     ┌──┐     ┌──┐
SDA:   │  │          │  │     │  │     │  │     │  │     │  │
       ┘  └──────────┘  └─────┘  └─────┘  └─────┘  └─────┘  └──────────
       │  │                      │        │        │        │           │
       START                     D7       D6       D1       D0        STOP
                                                      │
                                              ┌───────┘
                                              │ ACK (第9个时钟)
                                              │ 从机拉低 SDA
                                              ▼
                                          ┌──┐
                                    SDA:  │  0│  = ACK (应答)
                                          ┘   └──
                                          
    数据规则：SCL 低 → 允许改变 SDA (放数据)
             SCL 高 → SDA 必须保持稳定 (收数据)
```

**发送一字节：** SCL 低电平时主机将数据逐位放在 SDA 上（高位在前），拉高 SCL，从机在 SCL 高电平时读取。

**发送应答（SA）：** 收到数据后，在下一个时钟发送 0（应答）或 1（非应答）
**接收应答（RA）：** 发送数据后，在下一个时钟接收应答位

> **短记：** 谁收数据谁发 ACK（SA），谁发数据谁收 ACK（RA）。

### 9.4 AT24C02 EEPROM

- **存储介质：** EEPROM（电可擦除可编程只读存储器）
- **通信接口：** I²C
- **容量：** 256 字节（2Kbit）
- **功能：** 掉电不丢失，用于保存配置参数等持久化数据

**器件地址（本开发板）：**

```
固定地址 1010 + 可配置 A2A1A0 (000) = 1010000
写地址：0xA0
读地址：0xA1
```

**I²C 数据帧：**

```
起始 → 从机地址+R/W → ACK → 数据1 → ACK → ... → 停止
```

**复合格式（先写后读）：**

```
起始 → 从机地址+W → ACK → 寄存器地址 → ACK → 
起始 → 从机地址+R → ACK → 数据 → NAK → 停止
```

> **"帧"的类比：** 一帧图像 = 被边界框起来的一张图；一帧数据 = 被协议格式框起来的一包数据。帧意味着有边界、有格式、有意义的数据单元。

## 第十章：单总线与温度传感器

### 10.1 DS18B20 数字温度传感器

**特点：**
- 数字信号输入/输出（相比模拟传感器：功能强大、硬件简单、易扩展、抗干扰强）
- 测温范围：-55°C ~ +125°C
- 通信接口：1-Wire（单总线）
- 支持总线结构（一根线上挂多个传感器）
- 内置温度报警功能
- 支持寄生供电（只需 DQ + GND 两根线）

### 10.2 1-Wire 单总线

- 由 Dallas 公司开发
- **一根通信线** DQ
- 异步、半双工
- 最大的优点：**省线**

**操作流程：**
1. **初始化（复位）：** 主机拉低 ≥480μs，释放，从机拉低 60~240μs 应答
2. **ROM 操作：** ROM 指令 + 读写
3. **功能操作：** 功能指令 + 读写

### 10.3 1-Wire 时序

- 初始化：主机拉低 ≥480μs → 释放 → 等待 15~60μs → 从机应答 60~240μs
- 写 1：主机拉低 1~15μs → 释放 → 保持高电平 ≥60μs
- 写 0：主机拉低 ≥60μs → 释放
- 读位：主机拉低 1~15μs → 释放 → 在 15μs 内采样

> **FPGA 视角：** 1-Wire 的时序精度要求（微秒级）对单片机软件来说需要精心计算指令周期，但对 FPGA 而言则是轻松的 —— 一个 50MHz 的时钟周期是 20ns，480μs = 24000 个时钟周期，可以精确到 0.004%。这也是为什么工业中复杂时序控制任务倾向使用 FPGA。

### 10.4 温度数据格式

DS18B20 的温度值以 **16 位二进制补码**形式存储：
- 高字节：符号位 + 扩展位
- 低字节：温度值（低 4 位为小数部分）
- 读取后需要用公式转换为实际温度

### 11.1 蜂鸣器分类

### 11.2 驱动电路

蜂鸣器需要较大驱动电流，单片机 IO 口无法直接驱动，需要外加驱动：

**三极管驱动：**
- NPN：基极输入高电平导通，集电极输出（低端驱动）
- PNP：基极输入低电平导通，集电极输出（高端驱动）

**集成电路驱动（ULN2003）：**
- 达林顿晶体管阵列（7 路）

> **模电视角：达林顿管（Darlington）**
> 
> 两个三极管级联，第一个的输出电流驱动第二个，获得**极大的电流放大倍数**（β = β₁ × β₂）。
>
> | 对比 | 普通三极管 | 达林顿管 |
> |------|-----------|----------|
> | 电流放大倍数 | 较小（~100） | 很大（~10000） |
> | 导通压降 | ~0.7V | ~1.2~1.4V |
> | 开关速度 | 较快 | 较慢 |
> | 适用场景 | 一般放大/开关 | 大电流驱动 |
>
> 注意：单片机 IO 口默认上电后为**高电平**，蜂鸣器控制脚为 P2.5。

## 第十二章：AD/DA 转换

### 12.1 DAC（数字→模拟）

**从传感器到数字量的完整信号链：**

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 物理量    │    │  传感器   │    │  信号调理  │    │   ADC    │    │   MCU    │
│ 温度/光/  │───▶│ 热敏/光敏 │───▶│ 运放滤波  │───▶│ 逐次逼近  │───▶│ 数字处理  │
│ 压力/声音  │    │ 电阻/麦克风│    │ 放大/跟随  │    │ 12 bit   │    │ 显示/控制  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘

反向路径 (DAC):
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│   MCU    │    │   DAC    │    │  低通滤波  │    │  执行器   │
│ 数字值    │───▶│ T型/PWM  │───▶│ 平滑输出  │───▶│ 喇叭/电机  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

**主要实现方式：**

#### T 型电阻网络 DAC
利用运放和 R-2R 电阻网络，将数字量按位权相加转换为模拟电压。

#### PWM 型 DAC
PWM 信号 + 低通滤波器 → 模拟输出（利用占空比的积分平均值）。

**运算放大器的五种经典拓扑：**

```
1. 电压跟随器 (Buffer)          2. 反向放大器 (Inverting)
   ┌──────────┐                    ┌──────────┐
   │   ┌───┐  │                    │    Rf    │
   │ ┌─┤+  ├─┼── Vout             │  ┌──███──┐│
   │ │ │   │  │  Vout = Vin       │  │  ┌───┐││
   │ │ │   │  │                    │  ├──┤-  ├┼┼── Vout
   │ └─┤-  │  │                    │  │  │   │││  Vout = -Vin × (Rf/Rin)
   │   │   │  │                    │  │  │   │││
   │   └─┬─┘  │                    │ ┌┴──┤+  │││
   └─────┼────┘                    │ │   └───┘││
         └── Vout                  │ │        ││
                                   └─┴────────┴┘

3. 同相放大器 (Non-Inverting)    4. 差分放大器 (Difference)
   ┌──────────┐                    ┌──────────┐
   │    Rf    │                    │    R2    │
   │  ┌──███──┐│                    │  ┌──███──┐│
   │  │  ┌───┐││                    │  │  ┌───┐││
   │  ├──┤-  ├┼┼── Vout            │R1├──┤-  ├┼┼── Vout
   │  │  │   │││  Vout = Vin       │  │  │   │││  Vout = (V2-V1)
   │  │  │   │││  × (1 + Rf/Rg)   │ ┌┴──┤+  │││  × (R2/R1)
   │ ┌┴──┤+  │││                    │ │┌──┤   │││
   │ │┌──┤   │││                    │ ││  └───┘││
   └─┼┴──┴───┼┼┘                    └─┼┴───────┼┘
     │   Rg   │                       │        │
     └──███───┘                       V1       V2

5. 电压比较器 (Comparator)
   ┌──────────┐
   │   ┌───┐  │
   │V+─┤+  ├──┼── Vout
   │   │   │  │  V+ > V- → Vout = VCC
   │V-─┤-  │  │  V+ < V- → Vout = GND
   │   └───┘  │
   └──────────┘
```

> **模电视角：** 运放的"黄金法则"——**虚短**（两个输入端电压相等，V+ ≈ V-）和**虚断**（输入端不取电流，I+ ≈ I- ≈ 0）——只在**深度负反馈**条件下成立。电压比较器工作在开环状态，不满足虚短虚断条件。T 型电阻网络的巧妙之处在于：不管哪个开关接通，从任何节点看进去的等效电阻都是相同的，这使得权值精准可控。

### 12.2 ADC（模拟→数字）

**逐次逼近型 ADC** 最常用，核心原理是**二分查找**：

```
初始猜测值 = 满量程的一半
循环：
  if (DAC(猜测值) < 输入电压) → 增加
  else → 减少
  调整步长减半
直到 LSB 确定
```

### 12.3 XPT2046

- 集成 ADC 控制器
- 12 位分辨率，125KHz 转换速率
- 逐次逼近型
- SPI 通信接口（CS, DCLK, DIN, DOUT, BUSY）
- **上升沿输入，下降沿输出**

### 12.4 性能指标

| 指标 | 说明 |
|------|------|
| **分辨率** | 用位数表示（如 8bit = 256 级，12bit = 4096 级），位数越高越精细 |
| **转换速度** | 完成一次转换所需时间 |

### 12.5 常见传感器（需要通过 ADC 读取）

| 传感器 | 转换的物理量 |
|--------|-------------|
| 光敏电阻 | 光强 → 电阻 → 电压 → ADC → 数字量 |

### 13.3 PWM（脉冲宽度调制）

在具有惯性的系统中，通过调节脉冲宽度等效获取所需的模拟参数。常用于电机调速、开关电源等。

**PWM 参数：**

| 参数 | 定义 | 公式 |
|------|------|------|
| 频率 | 周期的倒数 | f = 1/Ts |
| 占空比 | 高电平时间占比 | D = Ton/Ts |
| 精度 | 占空比变化的最小步长 | 如 1%→2% 为 1% |

**产生 PWM 的方法：**
- 计数器 + 比较器：计数值与比较值比较，小于输出高、大于输出低
- 改变比较值 → 改变占空比

**PWM 原理波形图：**

```
计数器 ──────────────────────────────────────────────────
    CNT_MAX ┐                ┌────────┐
            │               ╱         │
            │              ╱          │
            │             ╱           │
            │            ╱            │
    比较值  │  ─  ─  ─ ╱─ ─  ─  ─  ─│─ ─  ─  占空比 = 比较值/CNT_MAX
            │         ╱               │
            │        ╱                │
            │       ╱                 │
            │      ╱                  │
      0     └────╱────────────────────└──────────▶ 时间
           ┌────┐    ┌──────────┐    ┌────┐
PWM 输出   │ 高  │    │    低     │    │ 高  │
           └────┘    └──────────┘    └────┘
           │←Ton→│←── Toff ──→│
           │←──────  Ts ──────────→│
           
           低通滤波后 ≈ 模拟电压 = VCC × (Ton/Ts)
```

> **计组视角：** PWM 控制的本质是"用时间换精度"——用一个只有高/低两种状态的数字信号，通过时间维度上的精确控制，在具有惯性（低通特性）的系统中等效出模拟量。这是数字控制模拟的经典范式。

**H 桥电机驱动原理：**

```
          VCC                    VCC
           │                      │
      ┌────┴────┐            ┌────┴────┐
      │   S1    │            │   S3    │
      │ (P-MOS) │            │ (P-MOS) │
      └────┬────┘            └────┬────┘
           │     ┌──────────┐     │
           └─────┤    M     ├─────┘
                 │  电机     │
           ┌─────┤          ├─────┐
           │     └──────────┘     │
      ┌────┴────┐            ┌────┴────┐
      │   S2    │            │   S4    │
      │ (N-MOS) │            │ (N-MOS) │
      └────┬────┘            └────┬────┘
           │                      │
          GND                    GND

   正转: S1 + S4 导通, S2 + S3 截止 → 电流从左到右
   反转: S2 + S3 导通, S1 + S4 截止 → 电流从右到左
   调速: 在 S1/S4 或 S2/S3 上施加 PWM
   刹车: S2 + S4 导通 (短路制动)
   
   注意：S1+S2 或 S3+S4 绝对不能同时导通！(直通短路)
         需要死区时间控制
```

## 第十五章：实时时钟 DS1302

### 15.1 芯片简介

DS1302 是由美国 DALLAS 公司推出的低功耗实时时钟芯片（RTC），具有：
- 计时功能：年、月、日、周、时、分、秒
- 闰年自动补偿
- 涓流充电能力（对备用电池充电）

**引脚：**
- VCC2：主电源
- VCC1：备用电源（接纽扣电池）
- GND：电源地
- X1、X2：接 32.768KHz 晶振
- CE：芯片使能
- I/O：数据输入/输出
- SCLK：串行时钟


### 15.2 BCD 码

DS1302 内部以 BCD 码（Binary Coded Decimal）存储时间数据。

**BCD 与十进制互转：**

```
DEC = BCD / 16 * 10 + BCD % 16
BCD = DEC / 10 * 16 + DEC % 10

示例：BCD 0x85 = 十进制 85
```

> **计组视角：** BCD 码是一种"以二进制编码十进制"的用码方案，每个十进制位独立存储在 4 个二进制位中。虽然浪费了部分存储空间（4bit 可表示 0~15，但只用 0~9），但避免了浮点运算中的舍入误差，在需要精确十进制表示的场景（时间、货币）中广泛使用。DS1302 选择 BCD 而非纯二进制存储时间，直接反映了嵌入式系统对"人机接口友好"的考量。

## 第十六章：存储器体系

### 16.1 存储器的本质模型

**存储器层次结构（Memory Hierarchy）：**

```
                    ┌─────────────────────────┐
       更快         │        寄存器            │  < 1ns,  ~128B,  CPU 内部
       更小         │       (Registers)        │
       更贵         ├─────────────────────────┤
        ▲           │        Cache            │  ~1ns,  ~KB,   SRAM
        │           │     (L1/L2/L3 Cache)     │     (51 单片机没有)
        │           ├─────────────────────────┤
        │           │        RAM              │  ~10ns, ~MB/GB, SRAM/DRAM
        │           │   (主存储器 / 内存)       │     51: 内部 256B SRAM
        │           ├─────────────────────────┤
        │           │    Flash / EEPROM        │  ~100ns, ~KB/GB
        │           │   (程序存储 / 固态存储)   │     51: 8KB Flash
        │           ├─────────────────────────┤
        ▼           │     HDD 机械硬盘          │  ~10ms, ~TB,   磁盘
       更慢         │   (磁介质 / 大容量)       │     (单片机通常没有)
       更大         └─────────────────────────┘
       更便宜

    访问时间对比：
    寄存器:    1 个 CPU 周期
    SRAM:      1~10 个 CPU 周期
    DRAM:      ~100 个 CPU 周期
    Flash:     ~1000 个 CPU 周期 (读) / ~100000 周期 (写)
    HDD:       ~10,000,000 个 CPU 周期
```

> **计组视角：** 存储器层次结构是计算机体系结构中最核心的设计理念之一。越靠近 CPU 的存储器速度越快、容量越小、单位成本越高。这个"金字塔"设计利用了**局部性原理**（时间局部性和空间局部性）来弥合 CPU 速度和主存速度之间的鸿沟。51 单片机虽然没有 Cache，但它的工作寄存器（R0~R7）和 SFR 可以看作是"第 0 层"的高速存储。

```
地址线 Address
     │
     ▼
┌──────────────┐
│  地址译码器   │ ← 根据地址选择对应的存储单元
└──────┬───────┘
       │
       ▼
┌──────────────────────────┐
│       存储单元阵列         │
│  [bit][bit]...[bit] ← 字0│
│  [bit][bit]...[bit] ← 字1│
└───────────┬──────────────┘
            │
            ▼
    ┌──────────────┐
    │  读写控制电路  │
    └──────────────┘
            │
   数据输出 / 数据输入
```

**三总线模型：**

| 总线 | 作用 |
|------|------|
| **地址总线** | 选择要访问哪个存储单元 |
| **数据总线** | 要读出或写入的数据 |
| **控制总线** | 决定读、写、使能或片选 |

**容量计算：**

```
存储容量 = 存储深度 × 数据位宽

示例：地址线 10 根 → 存储深度 = 2^10 = 1024
      数据位宽 = 8 bit
      总容量 = 1024 × 8 = 8192 bit = 1024 Byte = 1 KB
```

### 16.2 存储器分类

**存储器技术发展简史：**

```
年代     RAM 技术                    ROM/非易失技术
────     ────────                   ──────────────
1940s    汞延迟线 / 磁鼓            穿孔卡 / 纸带
1950s    磁芯存储器 (Core Memory)   ─
1960s    磁芯存储器                 Mask ROM (掩膜)
1970s    SRAM (Intel 1101, 256b)   PROM (熔丝, 一次性)
         DRAM (Intel 1103, 1Kb)    EPROM (紫外线擦除, Intel 1702)
1980s    DRAM (标准化, SIMM封装)    EEPROM (电擦除)
         SRAM (Cache 应用)          NOR Flash (Intel, 1988)
         PSRAM (伪静态RAM)          
1990s    SDRAM (同步DRAM, 66MHz)    NAND Flash (东芝, 1987 发明)

         RDRAM (Rambus, 短暂)
2000s    DDR2 (2003) → DDR3 (2007)  NOR Flash → 单片机程序存储
         Mobile DDR (LPDDR)         NAND Flash → SSD / U盘 / SD卡
2010s    DDR4 (2014) → LPDDR4      3D NAND (垂直堆叠, 三星 2013)

         HBM2E / HBM3              PLC (5-bit/cell) 研发中

  DIP 封装(古董) → SIMM 30pin → SIMM 72pin → DIMM 168pin(SDRAM)
  → DIMM 184pin(DDR) → DIMM 240pin(DDR2/DDR3) → DIMM 288pin(DDR4/DDR5)
  
  笔记本: SODIMM (Small Outline DIMM)

```

#### 易失性存储器（RAM，掉电丢失）

| 类型 | 存储原理 | 特点 | 用途 |
|------|----------|------|------|
| **SRAM** | 触发器/锁存结构（6 个晶体管） | 速度快、成本高、功耗高 | Cache、片上 RAM |
| **DRAM** | 电容存电荷（1 个晶体管 1 个电容） | 速度较慢、成本低、需刷新 | 电脑内存 DDR |

#### 非易失性存储器（ROM，掉电不丢失）

| 类型 | 可写次数 | 擦除方式 | 用途 |
|------|----------|----------|------|
| **Mask ROM** | 0 | 不可擦 | 大批量固化 |
| **PROM** | 1 次 | 不可擦 | 一次性编程 |
| **EPROM** | 多次 | 紫外线擦除 | 开发调试 |
| **EEPROM** | 多次（~100 万次） | 电擦除 | 参数保存（AT24C02） |
| **Flash** | 多次（~10 万次） | 电擦除（按扇区） | 程序存储、SSD |

#### Flash 细分

| 类型 | 架构 | 寻址 | 用途 |
|------|------|------|------|
| **NOR Flash** | 存储单元并联 | 可按字节寻址（支持 XIP） | 存 Bootloader、BIOS、单片机程序 |
| **NAND Flash** | 存储单元串联 | 按页/块读取 | SSD、U 盘、SD 卡 |

| 特性 | NOR Flash | NAND Flash |
|------|-----------|------------|
| 读取速度 | 极快（随机读） | 较慢（连续读尚可） |
| 写入/擦除速度 | 很慢 | 很快 |
| 容量 | 小（MB 级） | 大（GB/TB 级） |
| 每 MB 成本 | 高 | 低 |
| 芯片内执行（XIP） | **支持** | 不支持 |
| 坏块 | 几乎没有 | 存在，需 ECC |

#### SSD 内部结构

```
SSD = 主控芯片 + NAND Flash 颗粒 + 缓存 + 映射表（FTL）+ 固件 + 接口电路
```

#### Flash 单元类型

| 类型 | 每单元存储 bit 数 | 特点 |
|------|-------------------|------|
| SLC | 1 bit | 最快、寿命最长、最贵 |
| MLC | 2 bit | 性能和寿命均衡 |
| TLC | 3 bit | 当前主流，性价比较高 |
| QLC | 4 bit | 容量大、便宜、寿命较低 |


### 16.3 51 单片机的存储器映射

| 存储区域 | 作用 | 举例 |
|----------|------|------|
| Flash（程序存储器） | 存程序代码、常量表 | main 函数的机器码 |
| 内部 RAM（256B） | 存变量、栈、工作寄存器 | `unsigned char i;` |
| SFR（128B） | 控制 P0~P3、定时器、串口等 | `P0 = 0xFE;` |
| 外部 RAM（可选） | 扩展数据存储 | 普通实验一般不使用 |
| EEPROM（芯片内置或外挂） | 保存掉电不丢失的数据 | AT24C02 |

**程序运行模型对比：**

| 对比 | 单片机 | 电脑/Linux |
|------|--------|------------|
| 程序存放 | Flash/ROM | 硬盘/SSD |
| 加载方式 | 复位后 CPU 直接取指或 Bootloader 加载 | OS 加载到 RAM 执行 |
| 运行位置 | 常见是 Flash 取指（XIP） | 加载到 RAM 后执行 |
| 操作系统 | 可有可无 | 一般有 |
| 程序入口 | 复位向量/启动文件 | ELF 入口 + 启动代码 |

> **关键理解：** 程序不是一次性全部进入 CPU，而是存放在存储器中，CPU 靠 PC（程序计数器）一条一条取出来执行。这是冯·诺依曼/哈佛架构的基本执行模型。51 单片机采用**哈佛架构**，程序存储器和数据存储器地址空间分离，使用不同的指令访问（MOVC 读 Flash，MOV 读写 RAM）。

---

> **学习路线建议：**
> 1. 先掌握 LED/按键/数码管的基础 IO 操作，建立"代码控制硬件"的直观感受
> 2. 理解中断和定时器，这是单片机区别于纯软件编程的核心思维转变
> 3. 学习一种通信协议（推荐从 UART 开始），理解时序和协议分层
> 4. 逐步扩展到 I²C、SPI、1-Wire 等协议，对比它们的异同
> 5. 综合应用：将 LCD、传感器、电机、通信组合成完整的嵌入式系统
>
> 每一个外设的学习都可以沿着"硬件原理 → 寄存器配置 → 驱动封装 → 应用逻辑"的路径前进。
>
> 当遇到瓶颈时，尝试从计组（它为什么这样设计？）、数电（它内部电路怎么实现？）、模电（信号在物理世界怎么传递？）、FPGA（如果用纯硬件描述该怎么写？）的角度重新审视，往往会有更深的理解。
